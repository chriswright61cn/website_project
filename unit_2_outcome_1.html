1.1: Content and layout of the pages

The pages are split into sections based on the unit outcomes and each page is structured in a similar way. A fixed position video background covers the viewable screen. An image in a rounded box is positioned to the top right of the page.

A navigation bar, styled with flex-box runs across the top of the screen to access the different pages with the main heading page below it. A secondary fixed navigation block gives access to the targeted sections within the page.

The main content is positioned in a containing div. This holds the headings and paragraphs of the actual content. This content is styled with a transparent background to ensure visibility over the background image.

Fonts for styling are loaded from Google Fonts. A block is included for image attribution.

1.5: Copyright and other constraints affecting websites

The use of images and other content on a website depends on copyright and licensing. You cannot plagiarise other peoples content or use any image that you find on the web.

There are many websites that provide images or other content. Some are free for use on non commercial websites or require attribution. Others provide content free for use for any purpose. High quality stock content usually involves paying a royalty fee for it's use.

Commercial websites have stricter constraints on content. Many stock websites charge differently for commercial use of content.

The Companies Act 2006 requires that businesses disclose certain information about the identity of the company on their websites as well as terms and conditions that must be easily found. This does not have to appear on every page, but can be on an “About Us” page or in a footer section.

1.6: Accessibility

There are a number of ways to ensure that a website can be usable by as many people as possible. Making a website accessible to people with disabilities also benefits people using mobile devices, or those with slow network connections.

People with visual impairments such as blindness or poor vision can use a variety of accessibility tools. Screen readers read digital text out loud and screen magnifiers make text appear larger. The use of alt-tags to label images and using semantic HTML to organise your content, which improves accessibility for screen readers also improves Search Engines Optimisation.

People with colour blindness can be helped to access web content by paying attention to text and image contrast and not putting text over images.

People with mobility impairments may be unable to use a mouse and the website should be navigable by keyboard or an equivalent only, for example using the tab key to move between the different parts of a form.

People with cognitive impairments may have difficulty with understanding content or be confused by inconsistent or confusing page design. Consistent layout and navigation that everyone is familiar with will make websites easy to use.

1.7: Appropriate file-types

HTML is a text file with the markup for a websites content that tells web browsers how to display text, images and other media.

CSS is a text style sheet that defines how the html elements in an html page are displayed

Javascript is a script programming language that can interact with the html and css of the website.

JPG is a lossy bitmap graphics format suitable for displaying photographs with varied compression settings. It does not support transparency.

GIF is an old paletted bitmap graphics format with a limited number of colours that supports transparency. It is best used for images with large blocks of the same colour that compress easily. GIFs can be animated but there are now better options.

PNG is a bitmap graphics format with various sub-formats that can be paletted or not and supports transparency

WebP is a modern image format that provides superior lossless or lossy compression for images on the web. Lossless WebP supports transparency.

SVG is a vector graphics format that describes shapes using text. SVG images can be scaled up infinitely and elements within the file can be targeted and animated.

Various audio and video formats are supported including MP3, MP4, WebM, and Ogg.
