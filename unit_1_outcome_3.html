Unit 1, Outcome 3 (Review)

3.1: Review of the benefits and drawbacks of IT tools and systems used

The Image editor was fine for simple use but to expand the site with more images might require more advanced software to optimise and batch process image files

Prototyping software could have been used to design alternative interfaces and variations before writing any code. This would enable a designto be chosen before any markup is created and provide a plan of how to achieve the final site.

For this simple website it was possible to work alone but division of tasks would have made the project simpler.

The tasks could have been broken up into different areas of expertise such as content writing, adding content to html pages, web styling, image processing etc.

3.2: ways to improve productivity and efficiency

There are a number of ways that productivity could be improved. Using version control would enable trying out new ideas for the markup and styling solution. New ideas could be explored and reverted if not working

I could possibly have test pages with small amounts of content to test concepts for the interface before adding it to the main pages.

3.3: Develop solutions to improve my own productivity

If this web site project was to be repeated I would design a set of purely visual designs for the look and feel of the website. Once a final design was decided on then I could write markup and styling to conform to this.

Working as a team would have helped with productivity. Content could have been written while the design phase was being completed. This would enabled these tasks to be concentrated on without constantly changing the type of work being done. Then once the markup required has been added content could be dropped in.

Slack could help with this collaborative working. If channels are created for the different work areas then all the messages are visible. Finished content can be shared easily and anyone who joins the project can see what has been decided on and what work has been worked on and discussed. This help to review the progress of the project and see who is involved with the different aspects of the work.

3.4: Describe methods of testing digital solutions

Testing of the website by other users can reveal whether they find anything confusing on the site or hard to navigate. These problems can then be resolved.

Once all the content is added and the styling and menus are updated and tested I would test the website at different screen sizes to check that it works at different resolutions.

Once this step is complete I would then test it on different web browsers and devices to check for any other errors.

Automated testing software is available for website testing. Selenium is one solution that is open source and tests web applications across different browsers.
